1. Linux-Tools-Configure

1.1 Introduction

*inux对于做后端的工程师来说是会天天打交道的，理所应当的，Linux是 我们的首选；
Linux下的许多工具的使用是很讲究技巧性的，有很多黑技能能够使得 你的开发效率比别人快上几倍！
这里列出来，我日常经常会用到的工具，以及他的一些常规性的配置。

1.2 Tmux

tmux是我用过的第二个terminal工具，以前一直是screen，但是有一点是 screen并不能竖向切屏，而tmux提供更友好的切屏，所以我尝试去使用它。
在create windows方面上，两者的使用上也是没有什么很大的区别的，tmux可 以把prefix key变换成C-a。
在其他方面，我发现两者对于不同的终端模拟有着不同的展现，我自己 在使用secrecrt上使用xterm的方式时，screen表现正常，而对于tmux 却在刷屏上出现有些bug，对于自己是一个强迫症的人来说，这是不能容忍的。
在不换secrecrt，不换我那公司配的22寸的小屏幕的前提下， 目前我没有再继续使用tmux。但对于tmux的常用配置还是要了解的。
这里推荐几篇关于tmux的配置讲解的blog，个人感觉看了这几篇，就不用看我的了。

  • http://foocoder.com/blog/zhong-duan-huan-jing-zhi-tmux.html/
  • http://mingxinglai.com/cn/2012/09/tmux/

以下是自己的tmux配置，基本上是为了与screen保持一致的配置： 

##! BRIEF : tmux configure
##! AUTHOR: Aleda(aledalee@foxmail.com | aleda.cn)
##! DATE  : 2015-03-24 13:18
##! NOTICE: If impede your use, please change it and just take a mail to me.
##! [base configure]
##! 这里和你的终端模拟保持一致
set -g default-terminal "xterm"
set -g base-index 1
set -g pane-base-index 1
set -g status-utf8 on 
setw -g mode-keys vi
##! set pane
set -g pane-border-bg green
set -g pane-border-fg black
set -g pane-active-border-bg red
set -g pane-active-border-fg black
##! same as screen
set -g prefix C-a
unbind C-b
##! C-x - split window
unbind '"'
bind - splitw -v
##! C-x | vsplit window
unbind %
bind | splitw -h
##! C-x k,j,h,l select pane
bind k selectp -U
bind j selectp -D
bind h selectp -L
bind l selectp -R
##! C-x q kill pane
bind q killp
##! C-q kill window
bind C-q killw
##! C-a cycles choose windows
bind-key "C-a" select-window -t :-
##!  swap pane
##! swap with the previous pane (prefix Ctrl+u)
bind ^u swapp -U
##! swap with the next pane (prefix Ctrl+d)
bind ^d swapp -D
##! C-a A rename window
#bind A command-prompt -I #W "rename-window '%%'"
##! copy mode
##! enter copy mode (prefix Escape)
bind Escape copy-mode
##! paste buffer (prefix Ctrl+p)
bind ^p pasteb
##! select (v)
bind -t vi-copy v begin-selection
##! copy (y)
bind -t vi-copy y copy-selection
##! reload configure
bind r source ~/.tmux.conf \; display "Configuration reloaded!"

1.3 Screen

Screen的优点：

  • 多窗口，支持横向切屏
  • 会话共享，可以让一个或多个用户多次登录一个会话

缺点：

  • 4.00以后的版本才支持纵向切屏
  • 与sz rz的兼容性不好

官方说明：http://www.gnu.org/software/screen/
有兴趣的，还可以下源码下来看看。

给出一个详细的blog，我就不在这里班门弄斧了。
http://www.cnblogs.com/mchina/archive/2013/01/30/2880680.html

贴出我常用的配置： 

# Set default encoding using utf8
defutf8 on
## 解决中文乱码,这个要按需配置
defencoding utf8
encoding utf8 utf8
# 兼容shell 使得.bashrc .profile /etc/profile等里面的别名等设置生效
shell -$SHELL
#set the startup message
startup_message off
term linux
## 解决无法滚动
termcapinfo xterm|xterms|xs ti@:te=\E[2J
# 屏幕缓冲区行数
defscrollback 10000
# 下标签设置
hardstatus on
caption always "%{= kw}%-w%{= kG}%{+b}[%n %t]%{-b}%{= kw}%+w %=%d %M %0c %{g}%H%{-}"
#关闭闪屏
vbell off
#Keboard binding
# bind Alt+z to move to previous window
bindkey ^[z prev
# bind Alt+x to move to next window
bindkey ^[x next
# bind Alt`~= to screen0~12
bindkey "^[`" select 0
bindkey "^[1" select 1
bindkey "^[2" select 2
bindkey "^[3" select 3
bindkey "^[4" select 4
bindkey "^[5" select 5
bindkey "^[6" select 6
bindkey "^[7" select 7
bindkey "^[8" select 8
bindkey "^[9" select 9
bindkey "^[0" select 10
bindkey "^[-" select 11
bindkey "^[=" select 12
# bind F5 to create a new screen
bindkey -k k5 screen
# bind F6 to detach screen session (to background)
bindkey -k k6 detach
# bind F7 to kill current screen window
bindkey -k k7 kill
# bind F8 to rename current screen window
bindkey -k k8 title

1.4 Vim

Vim和Emacs向来都是*inux开发工程的两个阵营。两者都是神级的编辑器， 冒失点说，与顶级的IDE比起来，它也是不容小觑的。
这两个编辑器都各自有各自的有点，也正式因为两个阵营中，都有些顶级 的开发工程师在贡献着自己的劳动，所以两边用起来都非常不错。
我个人是两个都用过的，并且也在不断的尝试，但目前来说，自己还是比较 喜欢用Vim一些。 Vim的一些神级插件，自己也在不断的摸索。目前用到的一些插件如下:

  • NERDTree
  • TagList
  • vim-colors-solarized
  • DoxygenToolkit

这些详细的说明及使用，我就不在这里说了，在github上，一搜就出来了。
这里贴出我目前的vim配置: 

" @BRIEF  : vimrc配置文件
" @AUTHOR : Aleda(aledalee@foxmail.com | aleda.cn)
" @DATE   : 2015-03-28 23:05
" @VERSION: 1.0.0
" @USAGE  :
" ===================Vundle Begin=============================
set nocompatible              " be iMproved, required
filetype off                  " required
" set the runtime path to include Vundle and initialize
set rtp+=~/.vim/bundle/Vundle.vim
call vundle#begin()

" let Vundle manage Vundle, required
Plugin 'gmarik/Vundle.vim'
" solarized color
Plugin 'altercation/vim-colors-solarized'
" taglist
Plugin 'vim-scripts/taglist.vim'
" DoxygenToolkit
Plugin 'vim-scripts/DoxygenToolkit.vim'
" NERD Tree
Plugin 'scrooloose/nerdtree'

call vundle#end()
" ====================Vundle End==============================
syntax enable
" ==================Solarized Color Theme=====================
if has('gui_running')
    set background=light
else
    set background=dark
endif
colorscheme solarized
" ==================Solarized Color Theme=====================
"
" =======================Tag lists============================
" Taglist为最后一个window时,exit
let Tlist_Exit_OnlyWindow  = 1
" 在右边显示
let Tlist_Use_Right_Window = 1
" =======================Tag lists============================
"
" =====================DoxgenToolkit==========================
" 作者信息
let g:DoxygenToolkit_authorName="Aleda(aledalee@foxmail.com | aleda.cn)"
let g:DoxygenToolkit_blockHeader="--------------------------------------------------------------------------"
let g:DoxygenToolkit_blockFooter="----------------------------------------------------------------------------"
" =====================DoxgenToolkit==========================
"
" =====================Basic Configures=======================
syntax on
" 设置语法折叠
set foldmethod=syntax
" 启动Vim时候，默认关闭折叠

set nobackup
set smarttab
set tabstop=4
set shiftwidth=4

set autoindent
set cindent
retab
set nu
set hlsearch
set incsearch

set fileencoding=gb18030
set fenc=utf-8

set encoding=utf-8

" public auto indent

" 防止在vim方式下，鼠标选中默认成virtual
if has('mouse')

1.5 Emacs

学习和使用Emacs要经历陡峭的学习曲线，需要的是你不懈的使用和钻研。 对，它就想是一个不错的操作系统一样。
你可以上网搜索关于Emacs的不错教材，来进行学习和钻研，但最重要的是 你要有一颗平静的新去接受它，去不断的熟悉它。
